Java多线程

2020-05-12

1. 多线程概述

线程依赖于进程。

  1. 进程：正在运行的程序。是系统进行资源分配和调用的独立单位。每一个进程都有它自己的内存空间和系统资源。

  2. 多进程的意义：单进程的计算机只能做一件事，而现在的计算机都可以做多件事，即在一个时间段内执行多个任务，从而提高CPU的使用率。

    单CPU在一个时间点只能做一件事，CPU在程序间进行高效切换做到让我们觉得是同时进行的。

  3. 线程：在同一个进程内又可以执行多个任务，而这每一个任务就可以看成一个线程。线程是程序的执行单元（执行路径），是程序使用CPU的最基本单元。

    单线程：程序只有一条执行路径

    多线程：程序有多条执行路径

  4. 多线程的意义：提高应用程序的使用率。

    程序的执行其实就是在抢CPU的资源、执行权，某一进程线程比较多，就会有更高的几率抢到CPU的执行权。

    不能保证哪一个线程能够在哪个时刻抢到，所以线程的执行有随机性。

  5. 并发：逻辑上同时发生，指在某一个时间内同时运行多个程序

  6. 并行：物理上同时发生，指在某一个时间点同时运行多个程序

2. Java程序的运行原理

Java命令启动JVM，JVM启动就相当于启动了一个进程。由该进程创建一个主线程去调用main方法。

JVM虚拟机的启动是多线程的：垃圾回收线程也要先启动，否则很容易会出现内存溢出，即最低启动两个线程，所以JVM启动其实是多线程的。

4. 线程调度和优先级

线程只有得到CPU的时间片，也就是使用权，才可以执行指令。Java使用抢占式调度模型对线程进行调用。

  1. 两种调度模型：

    1. 分时调度模型：所有线程轮流使用CPU使用权，平均分配每个线程占用CPU时间片
    2. 抢占式调度模型：优先让优先级高的线程使用CPU，优先级相同则随机使选择一个。

  2. 优先级的设置

    1. int getPriority()：默认优先级为5
    2. setPriority(int newPriority)：最大为10，最小为1，默认5

    注意：线程优先级仅仅表示线程获取CPU时间片的几率，多次运行才能看出效果

6. 线程的生命周期

  3. 运行：有执行资格，有执行权

8. 线程安全问题

  1. 基本思想

    问题发生的原因：

    1. 是否是多线程环境：是
    2. 是否有共享数据：是
    3. 是否有多条语句操作共享数据：是

  2. 1、2无法改变，只能改变多条语句操作共享数据。

    思想：把多条语句操作共享数据的代码包成一个整体，让某个线程在执行时别人不能再执行。

    解决方法：

    同步代码块/同步方法

    • 格式：synchronized(对象)(需要同步的代码;)

      同步可以结局安全问题的根本原因就在对象上，该对象如同锁的功能。

    • 对象：任意对象都可以

    • 同步代码：可以是代码块，也可以是方法（将synchronized作为关键字加在方法上，private sychronized int sell(){}）。同步方法的锁对象为this。静态方法的锁对象为当前类的class对象（xxx.class）。

    • 同步的特点：
    1. 多个线程
    2. 多个线程使用同一个锁

    • 同步的好处：解决多线程的安全问题

    • 同步的弊端：降低程序执行效率（每个线程都去判断同步上的锁）

14. 线程池

程序启动一个新线程成本是比较高的，它涉及到与操作系统的交互。使用线程池可以很好地提高性能，尤其是当程序中要创建大量生存期很短的线程时，更应该考虑使用线程池。

线程池里的每一个代码结束后，并不会死亡，而是再次回到线程池中成为空闲状态，等待下一个对象来使用。JDK5后Java内置线程池。

Executors工厂类:

public static ExecutorService newSingleThreadExecutor()

ExecutorService对象表示一个线程池，可以执行Runnable或者Callable对象带代表的线程。

HTTP协议

2020-03-29

1. 概念

超文本传输协议，Hyper Text Transfer Protocol，定义了客户端和服务端通信时的传输协议。

特点：

  • 基于TCP/IP的高级协议
  • 默认端口号：80
  • 基于请求/响应模型：一次请求对应一次响应
  • 无状态协议：每次请求之间相互独立，不能交互数据

历史版本：

  • 1.0：每一次请求响应都会建立新的连接
  • 1.1：复用连接

2. 请求消息：客户端发送给服务器端的数据

  • 请求行

    请求方式 请求url 请求协议/版本

    GET /tomcatTest/haha.html HTTP/1.1

    • 请求方式

      • GET：

        1.请求参数在请求行中，在url后

        2.请求的url长度有限制

        3.不太安全

      • POST

        1.请求参数在请求体中

        2.请求的url长度没有限制

        3.相对安全

  • 请求头：客户端浏览器告诉服务器一些信息

    请求头名称：请求头值，请求头值

    常见的请求头：

    • User-Agent：浏览器告诉服务器，我访问你使用的浏览器版本信息

      可以在服务器端获得浏览器版本信息解决不兼容问题

    • Accept：浏览器可以接收的文件格式

    • Referer：http://localhost:8080/tomcatTest/haha.html

      告诉服务器我从哪里来，作用：

      1.防盗链

      2.统计工作

  • 请求空行

    空行，分隔POST请求头和请求体

  • 请求体(正文)：封装POST请求消息的请求参数

    name=value/text

3. 响应消息：服务器端发给客户端的数据

  • 响应行

    1. 组成：协议/版本 响应状态码 状态码描述

      HTTP/1.1 200 OK

    2. 响应状态码：服务告诉客户端浏览器本次请求和响应的一个状态

      1. 状态码都是3位数字
      2. 分类：
        1. 1xx：服务器接受客户端消息，但没有接受完成，等待一段时间后发送1xx状态码
        2. 2xx：成功。代表：200
        3. 3xx：重定向。代表：302（重定向），304（访问缓存）
        4. 4xx：客户端错误。代表：404（请求路径没有对应的资源），405（请求方式没有对应的doXXX方法）
        5. 5xx：服务器端错误。代表：500（服务器内部出现异常）

  • 响应头

    1. 格式： 头名称：值

    2. 常见的响应头：

      1. Content-Type：服务器告诉客户端本次响应体数据格式以及编码格式

      2. Content-disposition：服务器告诉客户端以什么格式打开响应体数据

        值：

        in-line：默认值，在当前页面打开

        attachment：filename=xxx：以附件形式打开响应体。如文件下载中使用

xml学习笔记

2020-03-23

1.xml的简介

XML是一种可扩展标记语言（eXtensible Markup Language)，是一种标记语言，类似于HTML，它被设计的宗旨是传输数据，而非显示数据。

标记型语言：如html，使用标签进行操作

可扩展：html中的标签是固定的，每个标签都有特定的含义

​ xml标签可以自己定义，甚至可以写中文的标签，如<猫></猫>

用途：HTML用于显示数据，XML也可以显示数据（不是主要功能）

​ XML主要功能是存储数据

版本：1.0（常用） 1.1

2.xml的应用

  • 不同系统之间的传输数据

  • 用来表示生活中有关系的数据

  • 经常用在文件配置

3.xml的语法

  • xml文档声明✨

    • 创建文件 后缀名是.xml

    • xml文件第一步必须要有一个文档声明(第一行第一列)，表示写xml文件的内容

      <?xml version="1.0" encoding="utf-8" ?>

      属性：version：xml的版本 1.0、1.1

      ​ encoding：xml的编码 gbk（中文简体）、utf-8、iso8859-1（不包含中文）

      ​ standalone：是否需要依赖其他文件 yes/no

      xml中文乱码问题？

      如保存到本地硬盘时使用gbk编码保存，打开时通过浏览器打开，使用设置的编码，此时就会出现乱码的问题。设置保存编码和打开编码一致即可避免乱码。

  • 定义元素（标签）✨

  • 定义属性✨

  • 注释✨

  • 特殊字符✨

  • CDATA区（了解）

  • PI指令（了解）

4. 元素的定义

  • 标签的定义有开始必须要有结束

    • 包含标签主体<person></person>
    • 不包含标签主体可在标签内结束 <aa/>

  • 标签的嵌套必须合理

  • 一个xml中只能有一个根标签，其他标签都是这个标签下面的标签

  • xml标签中出现的所有空格和换行都会被当成内容来解析

    <aa>aaaa</aa>

  • xml标签的名称规则：

    • 区分大小写（html不区分）

    • 不能以数字和下划线开头

    • 不能以xml、XML、Xml开头，如<xmla>

    • 不能包含空格、不能包含冒号（:）

5. xml属性的定义

  • html是标记型文档，可以有属性

  • xml也是标记型文档，可以有属性

  • 属性定义的要求

    • 一个标签上可以有多个属性

      <person id="id1" id2="id2"></person>

    • 属性名称不能相同

    • 命名规范和标签相同

    • 属性名称和属性值之间使用=，属性值使用单引号或双引号包起来

6. xml中的注释

<!-- 注释 -->

注释不能嵌套。

注释也不能放到第一列第一行，只能放文档声明。

7. xml中的特殊字符

如果想要在xml中显示a<b，不能正常显示，需要对特殊字符<进行转义：

&lt; <

&gt; >

&quot; “

&apos; ‘

&amp; &

8. CDATA区

解决多个字符都需要转义的操作：将内容放入CDATA区即不需要转义(把特殊字符当作文本内容而非标签)

<![CDATA[ 内容 ]]>

9. PI指令（处理指令）

可以在xml中设置样式。只能对英文标签名称起作用，对于中文的标签名称不起作用。

<?xml-stylesheet type="text/css" href="css文件路径"?>

XML语法总结：

  • 大小写敏感
  • 正确嵌套
  • 只有一个根元素
  • 属性值加引号
  • 转义字符必需转义或CDATA
  • 空格和回车换行解析时保留

10. XML的约束

对自定义的标签的约束。

xml约束的两种技术：dtd的约束和schema约束

  • dtd(一种简单的约束技术)

    • 创建一个文件 后缀名.dtd

    • 观察xml有多少个元素，有几个元素，在dtd文件中写几个<!ELEMENT>

    • 判断元素是简单元素（无子元素）还是复杂元素（有子元素）

      复杂元素：<!ELEMENT 元素名称 (子元素)>

      ​ 默认子元素只能出现一次，特殊符号表示子元素出现的次数：

      ​ <!ELEMENT student (name+,age?,sex*)

      ​ +:可以出现1次到多次

      ​ ?:出现0次或1次

      ​ *:可以出现0次到多次

      ​ 子元素使用逗号隔开表示元素的出现顺序，｜隔开表示只能出现这 些元素的任意一个。

      简单元素：<!ELEMENT 元素名称 (#PCDATA)>

      ​ (#PCDATA)：约束内容为字符串类型

      ​ EMPTY：约束内容为空

      ​ ANY：任意

    • 在xml文件中引入dtd文件

      • 引入外部dtd文件：

      <!DOCTYPE 根元素名称 SYSTEM "dtd的路径">

      • 内部标签

      • 使用外部dtd文件（网络上的dtd文件）

        <!DOCTYPE 根元素 PUBLIC "DTD名称" "DTD文档的URL">

    • 浏览器只负责校验语法而不负责校验约束，校验xml的约束需要使用工具。

  • Schema(复杂的约束技术)

    • Schema可以对xml标签元素的文本内容进行限定

    • 引入.xsd文档

      • 1.填写xml文档的根元素

      • 2.引入xsi前缀

      • 3.引入xsd文件的命名空间

        http://www.itcast.cn/xml即为2.xsd文件的命名空间，不声明前缀时需要在每个标签前都添加此约束。

      • 4.为每一个xsd的约束声明一个前缀

        xmlns="http://www.itcast.cn/xml"> 此时默认前缀为空

        xmlns:a="http://www.itcast.cn/xml">此时前缀为a，需要在每个使用2.xsd中限定的标签前都添加此前缀<a:name></a:name>

11. xml的解析

操作xml文档：

  • 解析（读取）：将文档中的数据读取到内存中
  • 写入：将内存中的数据保存到xml文档中，持久化数据

解析xml的方式：

  • DOM：将标记语言文档一次性加载进内存，在内存中形成dom树。（服务器端）

    优点：操作方便，可以将文档进行CRUD（增删改查）的所有操作

    缺点：消耗内存

  • SAX：逐行读取，基于事件驱动的。（移动端）

    优点：不占内存

    缺点：只能读取，不能增删改

12. xml常见的解析器

  • JAXP：sun公司提供的解析器，支持dom和sax两种思想
  • DOM4J：优秀的解析器，基于dom思想
  • JSoup：jsoup 是一款Java 的HTML解析器，可直接解析某个URL地址、HTML文本内容。它提供了一套非常省力的API，可通过DOM，CSS以及类似于jQuery的操作方法来取出和操作数据。
  • PULL：Android操作系统内置的解析器，sax方式的
